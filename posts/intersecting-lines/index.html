Intersecting Lines

Finding where two lines intersect is a pretty common task in computing.

Suppose we've got y1=2x+3 y_1 = 2x + 3 y1​=2x+3 and y2=−2x−2 y_2 = -2x - 2 y2​=−2x−2.

If we want to know where these two lines intersect, it becomes a question of where is y1=y2y_1 = y_2y1​=y2​.

We just set y1y1y1 equal to y2y2y2,

2x+3=−2x−2 , 2x + 3 = -2x - 2 \,,2x+3=−2x−2,

and solve for xxx.

After some algebra, we find that x=−54x = -\frac{5}{4}x=−45​.

using Plots

x = range(-5, 5, length=1000)

M = [ 2 ; -2 ]
B = [ 3 ; -2 ]
L1(x) = M[1]*x + B[1]
L2(x) = M[2]*x + B[2]

plot(x, L1.(x))
plot!(x, L2.(x))

From the graph, we can clearly see that −5/4-5/4−5/4 is the right answer.

To solve this programmatically we put the lines into standard form,

−2x+y=32x+y=−2 ,\begin{aligned} -2x + y &= 3 \\ 2x + y &= -2\,, \end{aligned}−2x+y2x+y​=3=−2,​

which we put into matrix form.

[−2121][x1x2]=[3−2]\begin{bmatrix}-2 & 1 \\ 2 & 1 \end{bmatrix}\begin{bmatrix}x_1 \\ x_2 \end{bmatrix} = \begin{bmatrix}3 \\ -2\end{bmatrix}[−22​11​][x1​x2​​]=[3−2​] 
using LinearAlgebra, RowEchelon

A = [ -2 1 ; 2 1 ]
b = [3 ; -2 ]

(x1, x2) = rref([ A b ])[:, 3]

@show x1, x2
> (x1, x2) = (-1.25, 0.5)

> (x1, x2) = (-1.25, 0.5)

If we don't have access to easy row reduction or can't generate the inverse of a matrix easily (which is actually really easy for R2\mathbb{R}^2R2 any way you cut it), the next best option is for us to use Cramer's Rule.

using LinearAlgebra

x1 = det([ b A[:, 2] ]) / det(A)
x2 = det([ A[:, 1] b ]) / det(A)

@show x1, x2
> (x1, x2) = (-1.25, 0.5)

And this is equivalent to,

(a1, a2) = A[:, 1]
(b1, b2) = A[:, 2]
(c1, c2) = b

d = a1*b2 - a2*b1

x1 = (c1*b2 - b1*c2)/d
x2 = (a1*c2 - c1*a2)/d

@show a1, a2
> (a1, a2) = (-2, 2)

@show b1, b2
> (b1, b2) = (1, 1)

@show c1, c2
> (c1, c2) = (3, -2)

@show x1, x2
> (x1, x2) = (-1.25, 0.5)
