Metric Tensors

When working with General Relativity, we naturally need to work with tensors. Fortunately, we can easily represent the metric, guvg_{uv}guv​, stress, and energy tensors using the Tensors package along with SymPy and LinearAlgebra.

Recall that a metric in general is essentially a kind of map we can use to determine how a coordinate system should be laid out.

The convention in GR, and more specifically when using Einstein summation notation, is to write the indices of contravariant vectors as superscripts and the indices of covariant vectors as subscripts. Read my posts on contravariant vs. covariant vectors and Einstein summation notation for a more in-depth treatment of each topic.

Starting from flatspace, we'll using the Pythagorean theorem and construct a metric tensor of rank 2.

ds2=(dx1)2+(dx2)2ds^2 = (dx^1)^2 + (dx^2)^2ds2=(dx1)2+(dx2)2

For a tensor transformation from the XXX frame of reference to the YYY frame of reference, we can use the following equation as a guide,

dXm=∂Xm∂YrdYr . \boxed{dX^m = \frac{\partial X^m}{\partial Y^r} dY^r}\,.dXm=∂Yr∂Xm​dYr​.

Applying (2)(2)(2) to a transformation of (1)(1)(1) will yield,

ds2=δmn∂Xm∂Yr∂Xn∂YsdYrdYs=grsdYrdYs .\begin{aligned} ds^2 &= \delta_{mn} \frac{\partial X^m}{\partial Y^r} \frac{\partial X^n}{\partial Y^s} dY^r dY^s \\ &= g_{rs}dY^r dY^s \,.\end{aligned}ds2​=δmn​∂Yr∂Xm​∂Ys∂Xn​dYrdYs=grs​dYrdYs.​

Before jumping to the metric tensor, gijg_{ij}gij​, where iii and jjj are indices, I think it's a little easier to understand from the perspective of using the Kronecker delta.

Looking at (1)(1)(1), we can see that we want to square the dx1dx^1dx1 and dx2dx^2dx2 terms. Furthermore, there are no terms with the product of dx1dx^1dx1 and dx2dx^2dx2, only dx1⋅dx1dx^1\cdot dx^1dx1⋅dx1 and dx2⋅dx2dx^2\cdot dx^2dx2⋅dx2.

Therefore, when m=n=1m = n = 1m=n=1, we will be squaring the dx1dx^1dx1 component and when m=n=2m = n = 2m=n=2, we will be squaring the dx2dx^2dx2 component. Since the Kronecker delta function returns 000 when mmm and nnn are different, whenever m≠nm \neq nm=n, those terms map to nullity. Therefore, the g11g_{11}g11​ term is 111, the g22g_{22}g22​ term is 111 and the g12=g21g_{12} = g_{21}g12​=g21​ term is 000.

In matrix form this is written as,

[g11g12g21g22] .\begin{bmatrix} g_{11} & g_{12} \\ g_{21} & g_{22} \end{bmatrix}\,.[g11​g21​​g12​g22​​].

To re-iterate the result with the coefficients emphasized, the metric is given by,

ds2=(1)(dx1)2+(0)dx1dx2+(1)(dx2)2 . ds^2 = (1)(dx^1)^2 + (0)dx^1 dx^2 + (1)(dx^2)^2\,.ds2=(1)(dx1)2+(0)dx1dx2+(1)(dx2)2.

Using the Tensors package, we can quickly construct a metric tensor for computation. We can use the Tensor or SymmetricTensor constructors. In either case we need to specify the tensor rank, the dimensions, and datatype.

Tensor{2, 2, Int}([1 0 ; 0 1])

We want a tensor of rank 222 with 222 dimensions, and a bit or numeric data type since we're working with 000s and 111s. In the example above, I specified the rank, dimension, and data type and then passed in a matrix of the form shown in (4)(4)(4). Another option would be to use the SymmetricTensor constructor. The difference here is that we are explicitly stating that our g12g_{12}g12​ and g21g_{21}g21​ terms are identical, thereby forming a symmetric matrix, so we can use the following syntax.

using Tensors, LinearAlgebra

g_cart = SymmetricTensor{2, 2, Int}((1, 0, 1))
>  2×2 SymmetricTensor{2, 2, Int64, 3}:
>   1  0
>   0  1

It's also worth noting that because of the symmetry here, we're only storing 333 data points instead of 444. While that's unlikely to make any difference with a 2−22-22−2 tensor (222 rank, 222 dimensional), it could offer improvements with higher-dimensional tensors.

Transforming the Metric Tensor to Other Coordinate Systems

We'll take it a step further. Let's perform a transformation from rectilinear coordinates to polar coordinates.

We'll need to leverage the following mappings,

x=rcos⁡θy=rsin⁡θ .\begin{aligned} x &= r\cos\theta \\ y &= r\sin\theta \,. \end{aligned}xy​=rcosθ=rsinθ.​

When we differentiate both xxx and yyy, we need to remember to use the chain rule since rrr is a function of θ\thetaθ.

dx=cos⁡θ dr−rsin⁡θ dθ dx = \cos{\theta}\ dr - r\sin{\theta}\ d\thetadx=cosθ dr−rsinθ dθ dy=sin⁡θ dr+rcos⁡θ dθ dy = \sin{\theta}\ dr + r\cos{\theta}\ d\thetady=sinθ dr+rcosθ dθ

Plugging (7)(7)(7) and (8)(8)(8) back into (1)(1)(1), expanding out, and collecting like terms yields the algebraical intensive but conceptually straight-forward,

ds2=(cos⁡θ dr−rsin⁡θ dθ)(cos⁡θ dr−rsin⁡θ dθ)+(sin⁡θ dr+rcos⁡θ dθ)(sin⁡θ dr+rcos⁡θ dθ)=cos⁡2θdr2−2rsin⁡θcos⁡θdr dθ+r2sin⁡2θdθ2+sin⁡2θdr2+2rsin⁡θcos⁡θdr dθ+r2cos⁡2θdθ2=cos⁡2θdr2+sin⁡2θdr2+r2sin⁡2θdθ2+r2cos⁡2θdθ2=(cos⁡2θ+sin⁡2θ)dr2+r2(sin⁡2θ+cos⁡2θ)dθ2=(1)dr2+(0)drdθ+(r2)dθ2 .\begin{aligned} ds^2 &= (\cos{\theta}\ dr - r\sin{\theta}\ d\theta)(\cos{\theta}\ dr - r\sin{\theta}\ d\theta) + (\sin{\theta}\ dr + r\cos{\theta}\ d\theta)(\sin{\theta}\ dr + r\cos{\theta}\ d\theta) \\ &= \cos^2\theta dr^2 - 2r\sin\theta\cos\theta dr\ d\theta + r^2\sin^2\theta d\theta^2 + \sin^2\theta dr^2 + 2r\sin\theta\cos\theta dr\ d\theta + r^2\cos^2\theta d\theta^2 \\ &= \cos^2\theta dr^2 + \sin^2\theta dr^2 + r^2\sin^2\theta d\theta^2 + r^2\cos^2\theta d\theta^2 \\ &= (\cos^2\theta + \sin^2\theta) dr^2 + r^2(\sin^2\theta + \cos^2\theta) d\theta^2 \\ &= (1)dr^2 + (0)drd\theta + (r^2)d\theta^2\,. \end{aligned}ds2​=(cosθ dr−rsinθ dθ)(cosθ dr−rsinθ dθ)+(sinθ dr+rcosθ dθ)(sinθ dr+rcosθ dθ)=cos2θdr2−2rsinθcosθdr dθ+r2sin2θdθ2+sin2θdr2+2rsinθcosθdr dθ+r2cos2θdθ2=cos2θdr2+sin2θdr2+r2sin2θdθ2+r2cos2θdθ2=(cos2θ+sin2θ)dr2+r2(sin2θ+cos2θ)dθ2=(1)dr2+(0)drdθ+(r2)dθ2.​

Note that above we used the trigonometric identity cos⁡2(θ)+sin⁡2(θ)=1\cos^2(\theta) + \sin^2(\theta) = 1cos2(θ)+sin2(θ)=1.

Now that we have simplified ds2ds^2ds2 to dr2+r2dθ2dr^2 + r^2 d\theta^2dr2+r2dθ2, we can construct our metric tensor of flatspace in polar coordinates.

grr=1grθ=gθr=0gθθ=r2\begin{aligned} g_{rr} &= 1 \\ g_{r\theta} &= g_{\theta r} = 0 \\ g_{\theta\theta} &= r^2 \end{aligned}grr​grθ​gθθ​​=1=gθr​=0=r2​

We could use functions here or represent the metric in a few other ways, but we'll leverage the excellent SymPy package and construct a symbolic tensor using the values from (10)(10)(10).

using SymPy, LinearAlgebra, Tensors

@vars r θ

g_polar = SymmetricTensor{2, 2, Sym}((1, 0, r^2))

g_polar yields a symbolic tensor of the form,

[100r2] .\begin{bmatrix} 1 & 0 \\ 0 & r^2 \end{bmatrix}\,.[10​0r2​].

Metric Tensor Using Spherical Coordinates

While I won't go through the full derivation here, it's worth offering the concrete example of how we create a tensor of rank 222 and dimension 333 constructed with the diagm function from the LinearAlgebra package.

After working through some extensive algebra, we will find ourselves with ds2ds^2ds2 in spherical coordinates,

ds2=(1)dr2+(r2)dϕ2+(r2sin⁡2θ)dθ2 . ds^2 = (1)dr^2 + (r^2)d\phi^2 + (r^2 \sin^2\theta)d\theta^2 \,.ds2=(1)dr2+(r2)dϕ2+(r2sin2θ)dθ2. 
using SymPy, LinearAlgebra, Tensors

@vars r θ ϕ

g_spherical = Tensor{2, 3, Sym}(diagm([1, r^2, r^2*sin(θ)^2]))

Notice that since we are working in dimension 333, we've changed the initial directive to 222 for rank 222, 333 for dimension 333, and once again we're using SymSymSym to leverage SymPy objects. Passed in as an argument is the output of the diagm function which constructs a diagonal matrix. For a 3×33 \times 33×3 matrix, diagm will expect a vector with 333 elements as shown above. More generally, for any n×nn \times nn×n matrix, it will expect a vector with nnn elements.

The assignment of g_spherical displays as,

[1000r2000r2sin⁡2(θ)] .\begin{bmatrix*}[c] 1 & 0 & 0 \\ 0 & r^2 & 0 \\ 0 & 0 & r^2 \sin^2(\theta) \end{bmatrix*}\,.⎣⎡​100​0r20​00r2sin2(θ)​⎦⎤​.
