Diophantine Equations

A Diophantine equation is simply a polynomial containing two or more unknowns with only integer solutions. Linear Diophantine equations have a form such as aX+bY=caX+bY=caX+bY=c while exponential Diophantine equations may have exponents such as XnX^nXn or YnY^nYn.

Diophantine equations are of great importance in cryptography and have a lot of interesting properties.

Suppose we have a linear Diophantine equation such as 2024X+748Y=C2024X + 748Y = C2024X+748Y=C. In order for 2024X+748Y2024X + 748Y2024X+748Y to have integer solutions, CCC must be some multiple of the greatest common divisor (GCD) of 202420242024 and 748748748.

Using the Euclidean algorithm, which we'll talk about in another post, we can compute that the gcd⁡(2024,748)=44\gcd(2024, 748) = 44gcd(2024,748)=44. This means that for 2024X+748Y=Q⋅gcd⁡(2024,748)2024X + 748Y = Q\cdot\gcd(2024, 748)2024X+748Y=Q⋅gcd(2024,748), there are integer values for XXX and YYY for every value of Q∈ZQ \in \mathbb{Z}Q∈Z.

This is a common programming exercise when teaching recursion.

function gcd(a, b)
    b == 0 ? a : gcd(b, a % b)
end

We can also write it more iteratively to match the computations that we will be doing manually.

function gcd(a, b)
    while b > 0
        q = div(a, b)
        r = a - q*b
        a = b
        b = r
    end
    return a
end

Thinking a bit more about this, we know that for 202420242024 and 748748748, the greatest common divisor is 444444. An interesting fact is that if the greatest common divisor of aaa and bbb happens to be 111, then aaa and bbb are co-prime or relatively prime meaning that the only number which divides both aaa and bbb is 111. What does this mean about the XXX and YYY solutions? We'll get back to that in a moment.

Thinking about 202420242024 and 748748748 again, we can use the "extended Euclidean algorithm" to determine that X=−7X = -7X=−7 and Y=19Y = 19Y=19 for the equation, 2024X+748Y=gcd⁡(2024,748)2024X + 748Y = \gcd(2024, 748)2024X+748Y=gcd(2024,748), therefore we know that,

2024⋅(−7)+748⋅(19)=44\begin{aligned} 2024 \cdot (-7) + 748 \cdot (19) = 44 \end{aligned}2024⋅(−7)+748⋅(19)=44​

We can continue scaling up or down the left and right hand sides of (1) and continue finding valid equations with integer solutions, such as

So, if we wanted to know what the solutions are for 2024X+748Y=882024X + 748Y = 882024X+748Y=88, since 2⋅44=882 \cdot 44 = 882⋅44=88, then we know that X=−7⋅2=−14X =-7\cdot 2 = -14X=−7⋅2=−14 and Y=19⋅2=38Y = 19\cdot 2 = 38Y=19⋅2=38 are solutions. Let's take this a step further with respect to finding solutions to other equations where the right hand side is given.

An Example with gcd⁡≥2\gcd \geq 2gcd≥2 and Solutions

Suppose we have the equation, 184X+72Y=32184X + 72Y = 32184X+72Y=32. Let's ask some questions.

  1. Does the equation have no integer solutions, one integer solution, or many integer solutions?

  2. If it has one or many, what are they?

The first thing we want to do is look at the equation,

184X+72Y=gcd⁡(184,72)\begin{aligned} 184X + 72Y = \gcd(184, 72) \end{aligned}184X+72Y=gcd(184,72)​

We start off by looking at 184=q⋅72+r184 = q \cdot 72 + r184=q⋅72+r and iteratively apply the division algorithm to find the greatest common divisor. The way we do this is by dividing 184184184 by 727272 which yields 2.555(555)2.555(555)2.555(555). We chop off the decimals and subtract 2⋅722 \cdot 722⋅72 from 184184184 to determine the remainder which in this case is 40.

Since we know that the greatest common divisor is 888, we can then backtrack to find values for XXX and YYY.

The important point here is that while we'll always be able to find a greatest common divisor for any non-zero integers, the reason we found a solution to the original equation is because the right hand side was a multiple of the GCD.

Using Julia

The Julia standard library comes with two functions, gcd and gcdx for the greatest common divisor and the extended Euclidean algorithm. gcd accepts two integers or an array of integers and will compute the gcd⁡\gcdgcd of all submitted values. The gcdx function takes aaa and bbb values and returns three values: the gcd⁡\gcdgcd, xxx and yyy.

# Solve AX + BY = K  =>  184X + 72Y = 32
a, b, k = 184, 72, 32
d, x, y = gcdx(a, b)

@show (x0, y0) = div(k, d) .* (x, y)
> (x0, y0) = div(k, d) .* (x, y) = (8, -20)

@show a*x0 + b*y0 == k
> a * x0 + b * y0 == k = true

Another example using Julia where we create functions for all possible solutions. We'll solve the equation, 9X+100Y=19X + 100Y = 19X+100Y=1.

a = 9; b = 100; k = 1;

function f(a, b, k)
    if gcd(a, b) == 1
        d, x0, y0 = gcdx(a, b)
        return t -> x0 + div(b, d)*t, t -> y0 - div(a, d)*t
    end
end

x, y = f(a, b, k)
sols = [ x.(-100:100) y.(-100:100) ]
all(x -> x == one(x), [a*s[1] + b*s[2] for s in eachrow(sols) ])
> true
