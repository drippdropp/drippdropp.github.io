Chinese Remainder Theorem

The Chinese Remainder Theorem is a tool for solving systems of modular equations.

Suppose we have a system of equations such as,

x≡a1(modn1)x≡a2(modn2)x≡a3(modn3)⋮x≡ak(modnk)\begin{aligned} x &\equiv a_1 \pmod{n_1} \\ x &\equiv a_2 \pmod{n_2} \\ x &\equiv a_3 \pmod{n_3} \\ &\vdots \\ x &\equiv a_k \pmod{n_k} \\ \end{aligned}xxxx​≡a1​(modn1​)≡a2​(modn2​)≡a3​(modn3​)⋮≡ak​(modnk​)​

If for all u,v∈Nu, v \in \mathbb{N}u,v∈N the gcd⁡(nu,nv)=1\gcd(n_u, n_v) = 1gcd(nu​,nv​)=1, then there's a unique solution for x(mod∏i=1kni)x \pmod{\prod_{i=1}^k n_i}x(mod∏i=1k​ni​).

Example

Suppose we have the three equations,

x≡2(mod3)x≡2(mod4)x≡1(mod5)\begin{aligned} x &\equiv 2 \pmod{3} \\ x &\equiv 2 \pmod{4} \\ x &\equiv 1 \pmod{5} \\ \end{aligned}xxx​≡2(mod3)≡2(mod4)≡1(mod5)​

and we want to solve for xxx. We know that xxx is going to be a combination of,

x≡x1(mod3)+x2(mod4)+x3(mod5) x \equiv x_1 \pmod{3} + x_2 \pmod{4} + x_3 \pmod{5} x≡x1​(mod3)+x2​(mod4)+x3​(mod5)

so we start off by adding some initial coefficients that will enable us to cancel out components modulo nin_ini​.

x≡5⋅4⋅x1+5⋅3⋅x2+4⋅3⋅x3≡20⋅x1+15⋅x2+12⋅x3 \begin{aligned} x &\equiv 5\cdot 4\cdot x_1 + 5\cdot 3\cdot x_2 + 4 \cdot 3 \cdot x_3 \\ &\equiv 20 \cdot x_1 + 15 \cdot x_2 + 12 \cdot x_3 \\ \end{aligned}x​≡5⋅4⋅x1​+5⋅3⋅x2​+4⋅3⋅x3​≡20⋅x1​+15⋅x2​+12⋅x3​​

Notice that since the last term is modulo 555, we add a factor of 555 to both the first and second terms. This will ensure that when we take the entire equation modulo 555, then those two terms will become 000, leaving only the third term. This only works because we know all of the nnn values are co-prime, meaning none of them are multiples of the others.

If we were to take x(mod3)x \pmod{3}x(mod3), then the x2x_2x2​ and x3x_3x3​ terms would go to 000, and 20≡2(mod3)20 \equiv 2 \pmod{3}20≡2(mod3), which fits. Next, we can scale 151515 by 222 since 30≡2(mod4)30 \equiv 2\pmod{4}30≡2(mod4). Finally, if we were to take x(mod5)x \pmod{5}x(mod5), the first two terms would now go to 000, but 12≡2(mod5)12 \equiv 2 \pmod{5}12≡2(mod5), so we need to find a multiple of 121212 which yields 1(mod5)1 \pmod{5}1(mod5). This is satisfied by scaling 121212 by 333 since 36≡1(mod5)36 \equiv 1\pmod{5}36≡1(mod5).

x≡20+30+36=86=26(mod60) x \equiv 20 + 30 + 36 = 86 = 26 \pmod{60} x≡20+30+36=86=26(mod60)

So, the basic idea is to try and isolate each component by scaling the coefficients such that the other terms will go to zero when we take some modulo kkk across all three. Ultimately, we then scale individual terms in order to find something that fits.

Now, this approach is how we can accomplish this by hand, but we incorporated quite a bit of inspection and a bit of trial and error to get the solution. In order to code a solver, we're going to need a step-by-step process which can be applied in the most general way.

It turns out that each componet can be computed with the following formula.

Let μi=∏i=1kni\mu_i = \prod_{i=1}^k n_iμi​=∏i=1k​ni​ and let ϕi\phi_iϕi​ be the modular inverse of μi\mu_iμi​ modulo nin_ini​. Therefore,

xi=ai⋅⌊μini⌋⋅ϕi , x_i = a_i \cdot \bigg\lfloor \frac{\mu_i}{n_i} \bigg\rfloor \cdot \phi_i \,,xi​=ai​⋅⌊ni​μi​​⌋⋅ϕi​,

and the final solution of xxx is,

x=∑i=1kxi . x = \sum_{i=1}^k x_i \,.x=i=1∑k​xi​.

Implementing this in Julia, we write a function, modsystemsolver, which accepts an array of aia_iai​ values and an array of nin_ini​ values,

modsystemsolver([a1⋮ak],[n1⋮nk])=q , \text{modsystemsolver}(\begin{bmatrix}a_1 \\ \vdots \\ a_k\end{bmatrix}, \begin{bmatrix}n_1 \\ \vdots \\ n_k\end{bmatrix}) = q\,,modsystemsolver(⎣⎡​a1​⋮ak​​⎦⎤​,⎣⎡​n1​⋮nk​​⎦⎤​)=q,

where qqq is the solution,

q=xmod  ∏i=1kni .q = x \mod{\prod_{i=1}^k n_i}\,.q=xmodi=1∏k​ni​.

In our implementation, we initially verify that all of the components of n are relatively prime to each other. We can leverage the gcd function in Julia's standard library which will take an array of integers and compute the greatest common divisor of all of them.

Next, we initialize the variable N with the product of the values of n. Then we iterate over the indices 1≤i≤∣n∣1 \leq i \leq |n|1≤i≤∣n∣, computing each xix_ixi​ value and using the reduce function to accumulate the sum. We finally return the result modulo NNN.

function modsystemsolver(a, n)
	gcd(n) > 1 && throw(
		ArgumentError("mod N values are not relatively prime"))

    N = prod(n)    
    result = reduce((acc, i) -> begin
        ai, ni = a[i], n[i]
        bi = fld(N, ni)
        
        acc + ai * bi * invmod(bi, ni)
        end, 1:length(n), init=zero(Int))
    mod(result, N)
end
