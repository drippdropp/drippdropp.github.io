Fermat Primality Test

Many classical primality tests are based on Fermat's Little Theorem which tells us that if,

ap−1≡1(modp) . a^{p-1} \equiv 1 \pmod{p} \,.ap−1≡1(modp).

where a∈Za \in \mathbb{Z}a∈Z, then ppp is prime, for all 1≤a<p1 \leq a < p1≤a<p.

Fermat Primality Test

Since there are more composites than there are primes, a common test is to randomly select some 1≤a<p1 \leq a < p1≤a<p and check to see if (1)(1)(1) is satisfied. However, there are some cases where the test will pass even though ppp is in fact composite. For example, 341=11⋅31341 = 11 \cdot 31341=11⋅31, so 341341341 is definitely composite. However, it will pass the Fermat Primality Test, for any aaa which is a power of 222, e.g. 2340≡1(mod341)2^{340} \equiv 1 \pmod{341}2340≡1(mod341). However, 341341341 will fail the test for other values of aaa. Another problem we have is that of Carmichael numbers which will actually pass for all values of aaa which are relatively prime to ppp even though ppp is a composite number. The smallest Carmichael number is 561561561.

We therefore have two problems to consider. The first is the problem of those composite numbers which will pass the primality test for some values of aaa.

In order to combat this problem, we repeat the Fermat primality test kkk times.

function fermatprimality(k, p)
    for _ in 1:k
        a = rand(1:p-1)
        res = mod(big(a)^(p-1), p) == 1
        if res == false
            return :composite
        end
    end
    return :likelyprime
end

fermatprimality(1, 341)
>  :likelyprime

We can run an experiment and plot how repetitions impact the validity of the primality test.

ks = 5
experimentcount = 10
results = zeros(Int, (experimentcount, 2, ks))
table = zeros(Float64, (ks, 2))

for k in 1:ks
    for i in 1:experimentcount
        res = [ fermatprimality(k, 341) for _ in 1:1000 ]
        c = filter(x -> x == :composite, res) |> length
        lp = 1000 - c
        results[i, :, k] = [c, lp]
    end
    table[k, :] = [
        mean(results[:, 1, k])/1000, 
        mean(results[:, 2, k])/1000
    ]
end

bar(table, label=["Composite" "Likely Prime"], xlabel="Test Repeats")

We can clearly see that by re-running the test a minimum of 555 times, we can reduce the potential of error to approximately 0.1%0.1\%0.1% which means this is a pretty good tool for verifying primality.

The second problem is that of Carmichael numbers. Fortunately, they are so few that we can manually check for a large chunk of them.
